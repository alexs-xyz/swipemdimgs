# swipemdimgs

Small GTK tool for reviewing image references in Markdown files generated by Docling.

Docling reference mode writes images into a sibling artifact directory such as:

```text
lecture.md
lecture_artifacts/
```

`swipemdimgs` opens each local Markdown image one after another so useless images can be marked for removal. Nothing is changed until the final confirmation.

## Usage

```bash
swipemdimgs path/to/file.md
```

For a non-destructive list of detected image references:

```bash
swipemdimgs --dry-run path/to/file.md
```

## Controls

- `k`: keep the current image
- `r`: mark the current image for removal
- `u`: undo and go back one image
- `Enter` / `y`: apply changes at the final confirmation and close
- `Esc` / `q`: cancel without changing anything

## What Apply Does

For every image marked with `r`, the tool:

1. Removes the Markdown line that references the image.
2. Deletes the image file if no remaining Markdown image reference uses that same file.

Image files are only deleted when they are inside the matching artifact directory:

```text
<markdown-stem>_artifacts/
<markdown-stem>_artefacts/
```

This keeps accidental references outside the Docling artifact folder from being deleted.

## Dependencies

The script uses Python 3 plus GTK 3 through PyGObject:

- `python3`
- `python-gobject`
- `gtk3`

On Arch/Omarchy these are system packages, not Python packages installed with `pip`.
